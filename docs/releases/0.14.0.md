# 影鉴 0.14.0 发布说明

- 发布日期：2026-07-20
- 发布类型：功能版本
- API 兼容性：现有 PortraitHub v1 推理、比对、人员库、任务和视频流接口保持兼容
- 控制台兼容性：需要重新构建并整体发布 Console Next 的 `dist/index.html` 与哈希静态资源

## 版本摘要

0.14.0 对照后端现有能力补齐 Console Next 的管理与业务操作面，新增受管成员与租户治理闭环，完善模型灰度发布、导出备份、Webhook、轨迹推理、视频任务、实时流、比对、检索和人员库高级参数，并统一列表分页与移动端布局。本版本同时启用“影鉴”原创品牌 Logo，并增加功能覆盖与版本一致性门禁。

## 身份、租户与接入治理

- 新增受管成员状态存储以及 `GET/POST/PATCH/DELETE /v1/admin/members`，支持按租户维护姓名、手机号、外部身份主体、角色和启停状态。
- 手机号统一规范为国际格式；成员角色必须来自服务端角色目录，空角色、非法角色和过长身份主体会稳定返回 422。
- OIDC 登录可按已验证手机号匹配预配置成员，并在首次登录时绑定外部 subject；已绑定 subject 不允许被其他身份抢占。
- 浏览器会话每次读取受管成员和租户状态。租户停用、成员停用或成员删除后，已有会话不再保留原角色权限。
- 租户支持名称和状态编辑，并展示成员、应用、Webhook 数量。
- 身份与权限页重构为“成员授权 / 租户管理 / 角色权限”三类工作区，支持成员创建、编辑、启停、移除以及租户创建、编辑、启停。
- 新建租户可同时创建默认接入应用，配置负责人、scope、每分钟限流、突发限流和每日配额；一次性密钥只展示一次。
- 接入应用表单补齐 JWT issuer、audience、限流和配额字段。
- Webhook 支持编辑、启停、失败重试次数和请求超时，保留密钥轮换与样例投递能力。
- 身份与租户写操作继续执行权限、租户隔离和审计；审计失败时恢复受管状态，避免出现未记录的权限变更。

## 模型发布与运维

- 模型中心新增模型详情、预热、运行时重载、加载/卸载和配置重载。
- 别名发布继续要求先预演再确认；新增多目标权重灰度发布、流量命中预览、回滚预演、确认回滚和发布审计。
- 修复权重灰度控件在半宽面板中越界的问题，桌面双栏和移动端单栏均不会相互遮挡。
- 运维页新增租户数据导出，支持增量起始时间以及人员、任务、视频流和单流事件上限。
- 备份创建支持 `updated_since` 增量起点，并在界面明确完整备份与增量备份的差异。
- 审计、备份、模型、评估数据集和阈值建议列表统一接入分页组件。
- 项目内 HTTP 422 校验统一使用固定 Starlette 版本支持的 `HTTP_422_UNPROCESSABLE_ENTITY`，避免不存在的状态常量把参数错误升级为 500。

## 推理、视频任务与实时流

- 图片分析新增“人员轨迹”入口，对接 `POST /v1/infer/tracks`，支持多图轨迹聚合、向量开关和检测参数。
- 人脸推理开放 `fallback_to_image`；人体、人脸和轨迹入口按后端契约传递置信度、IoU 和最大检测数。
- 视频任务新增类型/状态筛选以及检测模型、ReID 模型、置信度、IoU、最大目标数和向量开关。
- 实时流注册新增检测模型、ReID 模型、置信度、IoU、最大目标数、采样间隔、批次大小、读取超时和向量开关。
- 视频任务和实时流列表统一显示序号和分页；高级参数在窄屏自动切换为单列。

## 比对、检索与人员库

- 人员比对支持返回特征向量、批量异步提交和多模态融合参与项选择。
- 比对结果的融合明细和批量明细统一分页。
- 以图搜人可选择服务端阈值方案，不再固定使用单一阈值配置。
- 人员注册支持业务元数据键值对。
- 特征重建支持限定人体、人脸或衣着模态，并可指定目标模型；正式重建前必须完成同配置预演。
- 人员库列表统一分页，重建操作与 E2E 自动化同步遵守预演门禁。

## 前端一致性与品牌

- 新增共享 `DataTablePagination` 与 `useTablePagination`，统一列表序号、每页条数、翻页和服务端“加载下一批”衔接。
- 修复 Element Plus 分页 `small` 属性弃用警告，改用 `size="small"`。
- OpenAPI TypeScript 契约重新生成，覆盖成员管理、模型灰度、导出、轨迹推理及新增请求字段。
- 新增静态功能覆盖测试，阻止已补齐的后端能力从前端入口中意外回退。
- 新增“影鉴”原创 Logo：取景框表示视觉分析，人物轮廓表示身份画像，琥珀色扫描线表示识别核验。
- Logo 同时提供 SVG 与 512px 透明 PNG，并接入登录页、深色侧栏和浏览器 favicon。
- 390px 移动视口下，登录页、高级参数表单和分页组件无横向溢出。

## 升级与回退

1. 重新构建 Console Next，并整体发布 `dist/index.html`、`dist/.vite/manifest.json` 和全部哈希资源。
2. 清理或刷新 CDN/反向代理的 HTML 缓存；哈希资源可继续长期缓存。
3. 检查 OIDC 映射、预配置成员、租户状态和手机号唯一性，再开放企业账号登录。
4. 验证接入应用限流、Webhook 重试/超时、模型灰度预演和数据导出权限。
5. 验证 `/health` 与受保护的 `/ready/deep` 返回版本 `0.14.0`。
6. 回退时部署上一版完整镜像或受控静态构件，不删除成员、租户、审计、人员库或解析档案数据。
