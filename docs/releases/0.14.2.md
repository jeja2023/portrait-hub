# 影鉴 0.14.2 发布说明

- 发布日期：2026-07-21
- 发布类型：补丁版本
- API 兼容性：现有 PortraitHub v1 API 保持兼容
- 部署兼容性：现有 Docker 部署需要迁移可写模型配置路径并重建容器

## 版本摘要

0.14.2 修复异步视频 worker 只设置 `NVIDIA_VISIBLE_DEVICES`、却未在 Docker 默认 runtime 为 `runc` 时启用 NVIDIA runtime 的问题。流 worker 明确区分默认 CPU 模式和可选 GPU 模式。可写模型配置迁移到被 Git 忽略的 `runtime-state/models.yml`，避免模型中心写回后阻塞代码更新或回退。

## Docker 运行模式

### 异步视频 worker

`portrait-video-job-worker` 现在显式设置 `runtime: nvidia`，并继续通过 `VIDEO_JOB_WORKER_GPU_DEVICES` 控制可见设备。双 2080 Ti 默认配置为 GPU 0；容器启动后必须在容器内执行 `nvidia-smi` 验证设备注入。

### 实时流 worker

默认使用 CPU：

```dotenv
STREAM_WORKER_FORCE_CPU=true
STREAM_WORKER_GPU_DEVICES=none
```

需要使用 GPU 1 时改为：

```dotenv
STREAM_WORKER_FORCE_CPU=false
STREAM_WORKER_GPU_DEVICES=1
```

修改后必须通过 `docker compose up -d --force-recreate portrait-stream-worker` 重建容器，不能只执行 `restart`。

## 可写模型配置迁移

仓库根目录 `models.yml` 从运行时可写文件调整为初始化模板。Compose 默认挂载：

```dotenv
MODEL_CONFIG_HOST_FILE=./runtime-state/models.yml
MODEL_CONFIG_PATH=/workspace/models.yml
```

从 0.14.1 或更早版本升级时，在停止模型中心配置写入后执行：

```bash
cd /opt/portrait-hub
mkdir -p runtime-state
cp -a models.yml runtime-state/models.yml
cmp --silent models.yml runtime-state/models.yml
sed -i 's#^MODEL_CONFIG_HOST_FILE=.*#MODEL_CONFIG_HOST_FILE=./runtime-state/models.yml#' .env
git restore --source=HEAD -- models.yml
```

如果 `.env` 原先没有 `MODEL_CONFIG_HOST_FILE`，应手动追加该配置。后续更新不得使用仓库模板覆盖运行时副本。

## 升级步骤

1. 按 Ubuntu 教程备份 `.env`、`runtime-state`、数据库、对象存储和当前镜像 tar。
2. 完成上面的模型配置迁移，确认 `git status --short` 中不再出现 `models.yml`。
3. 拉取 0.14.2，执行 `docker compose -p portrait-hub config --quiet`。
4. 执行 `docker compose -p portrait-hub build --pull`。
5. 执行 `docker compose -p portrait-hub up -d --force-recreate --remove-orphans`，使全部服务切换到新的挂载和 runtime 配置。
6. 检查两个 API worker 的 `/ready/deep?load_models=true`，并在视频 worker 内检查 `nvidia-smi` 和 `CUDAExecutionProvider`。

## 回退步骤

1. 加载升级前保存的镜像归档，并切换到归档中记录的 Git SHA。
2. 保留 `MODEL_CONFIG_HOST_FILE=./runtime-state/models.yml`；旧 Compose 同样支持通过变量挂载该文件。
3. 执行 `docker compose -p portrait-hub up -d --no-build --force-recreate --remove-orphans`。
4. 仅在版本说明明确要求数据回退时恢复运行状态或外部数据，避免覆盖升级后产生的新数据。
