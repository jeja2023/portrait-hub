# 影鉴 0.14.1 发布说明

- 发布日期：2026-07-21
- 发布类型：补丁版本
- API 兼容性：现有 PortraitHub v1 推理、比对、人员库、任务和视频流接口保持兼容
- 控制台兼容性：需要重新构建并整体发布 Console Next 的 `dist/index.html`、manifest 和哈希静态资源

## 版本摘要

0.14.1 修复控制台退出后自动回到已登录页面的问题，并将模型中心的 GPU 选择从固定设备假设改为基于运行时可见设备的模型级配置。同时优化按权重灰度区域的版本角色选择器，使目标模型、流量权重和版本角色在桌面及窄屏下均清晰可读。

## 退出与会话行为

- 点击右上角退出后立即清理前端浏览器会话状态和权限缓存，按钮显示处理中并阻止重复点击。
- 服务端退出请求最多等待 3 秒；请求失败、超时或服务端会话已失效时仍会完成前端退出。
- 页面通过 `/?logged_out=1` 返回登录入口。登录页看到显式退出标记时不会静默调用浏览器会话恢复，因此不会再次进入控制台。
- API Key/JWT 高级凭证行为不变；退出只处理浏览器本地会话及 local/OIDC 服务端会话。

## GPU 调度配置

### 控制台能力

模型中心的模型列表显示运行时 GPU 清单和显存信息。每个模型都可以选择：

- 自动分配：不在模型配置中固定 `device_id`，由运行时按现有策略选择设备。
- 固定 GPU：从当前 worker 可分配设备中选择一个设备编号。

按权重灰度的模型目标现在按字段分组显示。版本角色选项为“当前稳定版本”和“候选灰度版本”，并在窄屏下自动切换为单列布局。

### API 与持久化

- `GET /v1/admin/models/gpu-devices` 返回配置设备、NVML 探测设备、探测来源和显存用量。
- `PATCH /v1/admin/models/{model_id}/gpu-device` 接受 `{ "device_id": 2 }` 或 `{ "device_id": null }`，分别代表固定设备和自动分配。
- 固定设备保存到目标模型的 `models.yml` 配置；更新时会卸载旧模型会话，下一次加载使用新配置。
- 设备不可见或不属于当前 worker 可分配范围时返回 400，不会修改配置。
- 每次成功变更写入 `model_gpu_device_updated` 审计事件，包含租户、请求、旧设备、新设备和分配模式。

### 部署边界

前端配置不能增加容器实际可见的 GPU，也不能改变 worker 数量。生产部署仍需先通过 `NVIDIA_VISIBLE_DEVICES`、Compose/Kubernetes GPU 资源和 worker 配置暴露设备。服务安装可选依赖 `requirements/prod-optional.txt` 中的 `nvidia-ml-py` 后，界面会显示 NVML 的实时显存；没有该依赖时仍可使用运行时配置清单。

## 升级步骤

1. 备份 `models.yml`、运行时状态目录和当前 Console Next 静态资源。
2. 更新后端镜像及 Console Next 静态资源，并确认所有运行时、npm workspace 和四种 SDK 版本均为 `0.14.1`。
3. 如果需要按 GPU 显存选择设备，安装 `requirements/prod-optional.txt` 中的可选 NVML 依赖，并重新启动 worker。
4. 进入模型中心，确认 GPU 清单与容器实际暴露设备一致，再为模型选择固定设备或保留自动分配。
5. 点击右上角退出，确认页面回到登录入口且不会自动重新进入控制台。
6. 进入模型中心的“发布”页，确认灰度目标显示为独立字段，版本角色下拉框显示“当前稳定版本/候选灰度版本”。

## 回退步骤

1. 回退后端镜像和 Console Next 静态资源到上一版本的完整构建产物，不要只回退 `index.html`。
2. 恢复备份的 `models.yml` 和运行时状态目录；如果新版本写入了 `device_id`，旧版本无法识别时应一并移除这些字段。
3. 清理 CDN/反向代理中的 HTML 缓存，保留哈希静态资源缓存。
4. 回退后验证 `/health`、受保护的 `/ready/deep`、登录、退出和模型加载状态。
