# 影鉴 0.16.0 发布说明

- 发布日期：2026-07-22
- 发布类型：体验优化版本
- API 兼容性：现有 PortraitHub v1 API 保持兼容
- 数据兼容性：不需要数据库或运行时状态迁移
- 部署兼容性：重新构建前端静态资源或 Docker 镜像即可

## 版本摘要

0.16.0 集中处理控制台导航和数据表的一致性体验：侧栏在较矮视口下不再显示垂直滚动条，底部入口选中态完整可见；全站数据表统一显示连续的序号；配置中心“全部配置”补齐单元格边框、分页和稳定的分类排序。

本版本不改变公开 API、权限模型、配置格式或后端数据结构。现有部署可以直接升级，生产环境只需重新构建并发布前端或镜像。

## 侧栏导航

- 桌面侧栏使用隐藏滚动条样式，仍支持滚轮、触控板和键盘滚动，不影响菜单访问。
- 桌面菜单项垂直间隔从 2px 收紧至 1px，给底部菜单保留更多可用高度，确保“运维与合规”选中边框不被面板底部裁切。
- 移动端原有横向导航行为保持不变。

## 数据表一致性

- 19 个原生 HTML 数据表和 4 个 Element Plus 数据表均包含“序号”列。
- 使用共享分页的列表以分页起始偏移计算序号，翻页后不会从 1 重新开始。
- 批量比对页清除重复的序号表头，保持列与数据一一对应。
- `tableSequence.test.ts` 会扫描表格模板并检查序号列，作为新增或修改列表时的回归门禁。

## 配置中心“全部配置”

入口：`系统管理 -> 配置中心 -> 全部配置`。

- Element Plus 表格启用 `border`，表头、单元格和行边界清晰可辨。
- 增加分页控件，默认每页 20 条，支持 10、20、50 条；切换筛选条件后自动回到有效页码。
- 桌面表格和移动端紧凑列表共用同一页数据与序号，避免两个视图显示数量不一致。
- 配置项按后端 `categories` 顺序排序；未知分类按中文名称排序，同分类内按配置键排序，筛选后仍保持相同顺序。

## 升级步骤

1. 备份 `.env`、`runtime-state`、模型文件和当前镜像。
2. 拉取 `0.16.0` 代码并确认版本一致性测试通过。
3. 重新构建 Console Next 或 Docker 镜像：

```bash
npm run console:build
docker compose -p portrait-hub up -d --build --force-recreate
```

4. 登录控制台，确认侧栏底部“运维与合规”选中态完整显示。
5. 打开任意列表和“系统管理 -> 配置中心 -> 全部配置”，确认序号、边框、分页及分类顺序。

本版本没有新增必须配置的环境变量，也不要求数据库迁移。现有 `.env` 和运行时状态文件可以继续使用。

## 回退

回退到 `0.15.0` 时，旧版本会忽略本版本新增的前端展示契约测试文件，但不会影响后端数据或运行时状态。回退前保留已发布的旧镜像，按现有 Docker 回退流程恢复前端和服务即可。
