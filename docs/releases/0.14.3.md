# 影鉴 0.14.3 发布说明

- 发布日期：2026-07-21
- 发布类型：补丁版本
- API 兼容性：现有 PortraitHub v1 API 保持兼容
- 安全兼容性：远程默认弱凭据仍会被拒绝，不降低本地管理员登录安全门槛
- 部署兼容性：使用 Docker 本地账号登录的部署需要复核 `.env` 并重建两个 API worker

## 版本摘要

0.14.3 针对服务器部署后登录页只显示“本地账号登录未配置”的问题，完善可见提示、环境模板、Ubuntu 排障命令和部署门禁。根因通常不是前端表单损坏，而是 `/v1/auth/config` 根据安全配置返回 `local_enabled=false`：Docker 网桥访问未开放、仍使用默认弱凭据、会话密钥不足 32 字节，或本地账号被关闭。

本版本同时包含 0.14.2 的视频 worker NVIDIA runtime、流 worker CPU/GPU 模式、`runtime-state/models.yml` 和镜像归档回退修复。

## 本地账号启用条件

Docker 或反向代理部署使用用户名和密码登录时，至少配置：

```dotenv
LOCAL_AUTH_ENABLED=true
LOCAL_AUTH_ALLOW_REMOTE=true
LOCAL_AUTH_USERNAME=admin
LOCAL_AUTH_PASSWORD=替换为强随机密码
LOCAL_AUTH_SESSION_SECRET=替换为至少32字节随机值
```

默认密码 `123456` 和默认会话密钥只能用于受限本机开发，不能与远程访问同时使用。生产 HTTPS 设置 `LOCAL_AUTH_COOKIE_SECURE=true`；直接 HTTP 或 SSH 隧道验收阶段使用 `false`。

## 升级步骤

1. 备份 `.env`、`runtime-state`、外部数据和当前镜像。
2. 使用 `openssl rand -hex 16` 生成本地管理员密码，使用 `openssl rand -hex 32` 生成独立会话密钥。
3. 更新 `.env` 中的 `LOCAL_AUTH_*` 配置，不要复用 API Token、OIDC 或数据加密密钥。
4. 拉取并构建 0.14.3，然后强制重建两个 API worker：

```bash
docker compose -p portrait-hub up -d --build \
  --force-recreate gpu-worker-0 gpu-worker-1
```

5. 分别验证：

```bash
curl --fail --silent http://127.0.0.1:9001/v1/auth/config
curl --fail --silent http://127.0.0.1:9002/v1/auth/config
```

两个响应都必须包含 `"local_enabled":true`。随后刷新登录页，确认显示用户名和密码表单并完成登录、退出回归。

从 0.14.1 或更早版本直接升级时，还必须执行 0.14.2 发布说明中的 `runtime-state/models.yml` 迁移。

## 诊断与回退

登录入口仍不可用时，使用 Ubuntu 教程第 20.7 节的容器内脱敏诊断命令。不要通过恢复默认密码或缩短会话密钥绕过安全检查。

回退时加载升级前镜像、切换对应 Git SHA，并使用 `--no-build --force-recreate` 重建容器。若保留 0.14.3 的强 `LOCAL_AUTH_*` 配置，旧版本仍可继续使用本地账号登录。
