# 影鉴 0.15.0 发布说明

- 发布日期：2026-07-21
- 发布类型：功能版本
- API 兼容性：现有 PortraitHub v1 API 保持兼容，新增管理员配置接口
- 部署兼容性：现有 `.env` 可继续启动；升级后需要重新构建镜像和重建四个服务

## 版本摘要

0.15.0 新增统一配置中心，将原来分散在 `.env` 中的项目配置集中展示和管理。配置目录直接读取随镜像发布的 `.env.example`，新增配置项会自动进入页面；管理员可以按分类、关键词和生效方式检索，查看当前值来源，并创建或移除配置覆盖。

视频流和 Webhook 网络访问策略独立支持单 IP、域名、IPv4 CIDR 和 IPv6 CIDR，适用于无互联网、无内部域名且摄像头数量较多的局域网。网络策略保存后由 API 与 Worker 从共享状态读取，不需要重建容器。

配置中心在桌面端使用可扫描表格，在移动端改为独立紧凑列表；底部导航支持跨工作台、开发者中心和系统管理横向访问，并自动将当前页面入口滚动到可见区域。

## 配置中心

入口：`系统管理 -> 配置中心`。

新增接口：

- `GET/PUT /v1/admin/configuration`
- `GET/PUT /v1/admin/network-access-policy`

接口要求 `admin:configuration` 权限；内置管理员角色通过通配权限访问，其他角色默认无权读取或修改系统配置。

配置覆盖保存在 `runtime-state/admin-configuration.json`。文件权限会尽力限制为 `0600`，敏感值只允许替换，不通过读取接口、页面或审计事件回显。状态文件损坏时遵循 `STATE_READ_FAIL_CLOSED`，默认拒绝静默忽略。

配置按生效方式分为：

| 类型 | 生效方式 |
| --- | --- |
| 视频流和 Webhook 网络策略 | 保存后立即生效 |
| FastAPI、鉴权、限流、存储客户端等应用配置 | 重启相关服务后生效 |
| GPU 设备、宿主机挂载、构建参数和 worker 模式 | 同步宿主机 `.env` 后重建容器 |

## 局域网 CIDR 策略

无域名摄像头可以按独立 VLAN 配置：

```text
10.30.0.0/16
```

Webhook 服务可使用独立网段：

```text
10.40.0.0/24
```

启用私网访问时，服务端强制要求至少配置一个主机或 CIDR。规则标准化后去重保存，支持并发修订号检查，并在真正连接视频流前再次校验。

保存时会在写锁内重新读取共享状态的最新修订号，拒绝并发管理员提交的过期版本。单 IP 规则只允许精确匹配；只有域名规则允许匹配其子域，避免数字 IP 被误作域名后缀。

## Docker 编排配置

配置中心保存 Docker 编排目标值后，在宿主机执行：

```bash
python tools/apply_admin_configuration.py --check
python tools/apply_admin_configuration.py
docker compose -p portrait-hub up -d --force-recreate
```

工具只同步允许的编排项，写入前备份 `.env`，不会在终端输出配置值。应用运行配置只需要重启：

```bash
docker compose -p portrait-hub restart \
  gpu-worker-0 gpu-worker-1 portrait-video-job-worker portrait-stream-worker
```

## 升级步骤

1. 备份 `.env`、`runtime-state`、模型文件和当前镜像。
2. 拉取 0.15.0 代码。
3. 重新构建并重建四个服务：

```bash
docker compose -p portrait-hub up -d --build --force-recreate
```

4. 登录后进入配置中心，设置摄像头和 Webhook 的允许网段。
5. 验证两个 API worker、视频任务 worker 和视频流 worker 状态。

现有 `.env` 不要求立即加入 `STREAM_ALLOWED_CIDRS`、`WEBHOOK_ALLOWED_CIDRS` 或 `ADMIN_CONFIG_STATE_PATH`；未配置时使用共享运行状态目录下的默认文件。

## 回退

回退到 0.14.3 时，旧版本会忽略两个新增状态文件。配置中心中保存的应用覆盖和 CIDR 策略不会在旧版本生效，因此回退前应确认 `.env` 仍包含旧版本所需的关键配置。不要删除新增状态文件，以便重新升级后恢复配置。
